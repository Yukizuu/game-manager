# 制作ワークフロー

## 素材管理

素材は `assets` ディレクトリで管理します。

画像、音声、UI、参考資料などは用途ごとに整理し、どのゲームで使う素材か分かるようにします。

## ゲームでの利用

ゲーム側で素材を使う場合は、`assets` から必要なファイルをローカルコピーして使用します。

ゲームごとの実行環境やビルド環境に必要な形でコピーし、ゲーム本体から参照できる場所に配置します。

## GitHub画像の扱い

ゲーム内ではGitHub上の画像URLを直接参照しません。

GitHubの画像を直接参照すると、ネットワーク状況やリポジトリ構成の変更により表示できなくなる可能性があります。

ゲームで使用する素材は、必ずローカルに配置したファイルを参照します。
